# EasyColoc 优化目标与开发路线图

## 1. 核心定位
**项目愿景：** 为多组学共定位分析提供一站式、高自动化、兼容多类型 QTL 的工作流。用户仅需提供 GWAS 与 QTL 汇总数据（summary statistics），EasyColoc 将自动完成变异匹配、多信号识别等流程，并输出学术级报告。

---

## 2. 架构设计：兼容多类型 QTL
为了提升软件的泛用性与易用性，需在数据解析层进行充分解耦：

* **标准化输入配置:** 引入通用参数或 YAML 配置来定义列名映射。无论是 eQTL (Gene)、sQTL (Junction) 还是 caQTL/mQTL (Peak)，均统一抽象为基础计算单元 `phenotype_id`。
* **智能表型处理策略:**
    * **Gene-level:** 默认的一对一共定位计算。
    * **Multi-feature level:** 针对同一区域内存在的多个剪接位点或 APA 位点，提供“独立并行计算”或“合并计算”的灵活选项。
* **坐标系版本支持:** 内置对 GRCh37/38 参考基因组的自动识别及 LiftOver 转换适配功能。

---

## 3. 面对现有痛点的优化策略

| 功能模块 | 常见工具痛点 | **EasyColoc 优化方案** |
| :--- | :--- | :--- |
| **数据接入** | 格式要求严格，不同类型 QTL 需频繁的数据预处理代码。 | **灵活解析器**：内置常用 QTL 数据库（如 GTEx, eQTLGen 等）格式的一键读取支持机制。 |
| **变异对齐** | rsID 缺失或基点不一致极易导致流程发生异常中断。 | **多级匹配引擎**：基于坐标与等位基因构建多维度兜底匹配逻辑（如优化后的 `identify_loci`）。 |
| **精细定位** | 大多基于单因果假设，在强 LD 区域容易产生混淆和假阳性。 | **集成 SuSiE-RSS**：实现多因果变异精细定位（fine-mapping），解析真正的独立信号。 |
| **结果展示** | 仅输出 CSV/TSV 结果表，后续报告绘制和可视化的学习成本高。 | **自动化报告**：一键生成包含共定位概览、LocusZoom、效应量相关性的可视化反馈。 |

---

## 4. 技术实现难点

### A. 变异对齐与匹配
* **多级匹配逻辑:** 匹配优先级设定为 `rsID` -> `chr:pos:ref:alt` -> `chr:pos`。
* **等位基因一致性检查:** 严格比对 GWAS 和 QTL 的 risk allele。如发现反向，自动 flip 效应量（Beta/Z-score），减少可用位点的被动丢失。
* **LD Clumping 降级机制:** 当 PLINK 因外部 LD reference 匹配率过差无法运行时，降级采用基于物理距离的 Pruning，并妥善记录日志予以提示。

### B. 统计模型层
* **多信号提取:** 针对 MHC 等高密度的复杂 LD 区域，引入 `coloc.susie` 支持的多信号共定位模型。
* **敏感度分析评估:** 计算不同先验概率设定对最终共定位概率 (PP4) 的影响，通过敏感度曲线证明核心结果的稳健度。

---

## 5. 输出格式与全流部署

### A. 分析报告
* **汇总表格 (Summary Table):** 整理并高亮显著信号关联的基因、组织、PP4 得分，及是否支持多独立信号。
* **综合图表展示:** 自动叠合 GWAS 和 QTL 信号的 LocusZoom 堆叠散点图，以及效应强度的双向相关性图。

### B. 系统部署
* **Docker 部署方案:** 打包 R 语言环境 (包含 susieR, coloc 包) 及 Python 依赖，提供“开箱即用”的使用体验。
* **提供测试数据集:** 设计测试数据 (Dummy Data) Demo，以便用户验证软硬件环境。

---

## 6. 开发周期分配 (Milestones)

1. **Phase 1 (对齐模块):** 完善并测试 `identify_loci` 逻辑，处理各种极端情况的 SNP 匹配。
2. **Phase 2 (输入兼容):** 设计灵活的表型参数及通用列名映射配置文件。
3. **Phase 3 (算法支持):** 测试和集成多信号定位算法框架 (`coloc.susie`) 作为进阶选项。
4. **Phase 4 (结果展示):** 引入 RMarkdown 及对应绘图包，实现标准化的交互式数据报告流。
